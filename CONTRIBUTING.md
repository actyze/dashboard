# Contributing to Dashboard

## Documentation Standards

To keep the project organized, please follow these rules for documentation:

1.  **Docker Documentation**: All documentation related to Docker deployment, configuration, or usage must be placed in the `docker/` directory.
2.  **Helm/Kubernetes Documentation**: All documentation related to Helm charts, Kubernetes deployment, or configuration must be placed in the `helm/` directory.
3.  **General Documentation**: High-level architecture, setup guides for external services, and other general documentation can remain in the root directory.

## Project Structure

- `docker/`: Contains Docker Compose files and Docker-specific documentation (e.g., `DEPLOYMENT.md`).
- `helm/`: Contains Helm charts and Helm-specific documentation (e.g., `DEPLOYMENT.md`, `VALUES_README.md`).
