# Query History Enhancement

## Overview
Comprehensive query history tracking with support for both natural language and manual SQL queries, including LLM metadata, chart recommendations, and user-provided names.

## Database Changes

### New Columns Added to `nexus.query_history`

| Column | Type | Description |
|--------|------|-------------|
| `query_name` | VARCHAR(255) | User-provided name for the query (optional, can be set/updated later) |
| `query_type` | VARCHAR(20) | Type of query: `natural_language` or `manual` |
| `chart_recommendation` | JSONB | LLM chart recommendation: `{chart_type, x_column, y_column, title}` |
| `llm_response_time_ms` | INTEGER | Time taken by LLM to generate SQL (milliseconds) |
| `generated_at` | TIMESTAMP | When SQL was generated by LLM or entered manually |
| `executed_at` | TIMESTAMP | When SQL was executed against Trino |
| `updated_at` | TIMESTAMP | Last update timestamp (auto-updated on changes) |

### Migration Script
- **File**: `helm/dashboard/sql/05-query-history-enhancement.sql`
- **Features**:
  - Adds all new columns with proper constraints
  - Backfills `generated_at` and `executed_at` for existing records
  - Creates indexes for performance
  - Adds helpful views: `query_history_with_users`, `query_history_summary`
  - Includes helper function `generate_query_title()` for auto-titles

## API Endpoints

### 1. Get Query History
```http
GET /api/query-history?limit=50&offset=0&query_type=natural_language
Authorization: Bearer <token>
```

**Response:**
```json
{
  "success": true,
  "queries": [
    {
      "id": 123,
      "query_name": "Top Customers Analysis",
      "query_type": "natural_language",
      "natural_language_query": "show me top 10 customers by sales",
      "generated_sql": "SELECT ...",
      "execution_status": "success",
      "execution_time_ms": 250,
      "llm_response_time_ms": 1500,
      "row_count": 10,
      "chart_recommendation": {
        "chart_type": "bar",
        "x_column": "customer_name",
        "y_column": "total_sales"
      },
      "generated_at": "2025-12-06T18:30:00Z",
      "executed_at": "2025-12-06T18:30:01Z",
      "created_at": "2025-12-06T18:30:01Z"
    }
  ]
}
```

### 2. Update Query Name
```http
PATCH /api/query-history/{query_id}/name
Authorization: Bearer <token>
Content-Type: application/json

{
  "query_name": "My Custom Query Name"
}
```

### 3. Save Manual Query
```http
POST /api/query-history/manual
Authorization: Bearer <token>
Content-Type: application/json

{
  "sql": "SELECT * FROM customers LIMIT 10",
  "max_results": 500,
  "timeout_seconds": 30
}
```

## Usage Examples

### Natural Language Query (Auto-saved)
```python
# Frontend calls /api/generate-sql
# Backend automatically saves with:
# - query_type: 'natural_language'
# - chart_recommendation: {...} (from LLM)
# - llm_response_time_ms: <time>
# - generated_at, executed_at: <timestamps>
```

### Manual SQL Query
```python
# Frontend calls /api/query-history/manual
# Backend saves with:
# - query_type: 'manual'
# - query_name: Auto-generated title (e.g., "Query from customers")
# - generated_at, executed_at: <current time>
```

### Update Query Name
```python
# User clicks "Rename" in UI
# Frontend calls PATCH /api/query-history/123/name
# Backend updates query_name and updated_at
```

## Database Views

### `nexus.query_history_with_users`
Joins query history with user details for easy querying.

### `nexus.query_history_summary`
Lightweight view with truncated SQL for listing/search.

## Helper Functions

### `nexus.generate_query_title(sql_text TEXT)`
Auto-generates descriptive titles based on SQL operation:
- `"Count from customers"` for COUNT queries
- `"Sum from orders"` for SUM queries
- `"Query from table_name"` for SELECT queries
